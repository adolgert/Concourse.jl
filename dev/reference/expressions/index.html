Expression algebra

Every random quantity in a model — interarrival times, service times, patience, marks — is declared as a law written in this small algebra. A law is data, not a closure, so compile can see exactly which parameters, marks, and times each law reads and check the model statically. The surface language consumes these values, and the IR evaluates them when clocks are enabled.

Expressions

Concourse.ScalarExpr — Type
ScalarExpr

Abstract supertype of the expression algebra that law arguments are written in. A ScalarExpr is a value, not a closure, so the compile-time checker can see what an expression reads: parameters (reads_params), marks (reads_marks), time (reads_time), and station occupancy (reads_state).

The leaves are Param, Mark, Enab, Const, InService, and InBuffer. Expressions combine with +, -, *, /, inv, log, exp, sqrt, min, max, ceil, and floor. A bare number in arithmetic with an expression is promoted to a Const. The staircase functions ceil and floor are flat almost everywhere, so a parameter inside their argument contributes no pathwise derivative — genuinely, not as an approximation (a dual number entering them leaves as a constant).

Example

julia> rate = Param(:lambda) * 2;

julia> reads_params(rate)
Set{Symbol} with 1 element:
  :lambda
source
Concourse.Param — Type
Param(name::Symbol)

The model parameter called name. Evaluation reads the entry of the parameter vector θ at the position compile assigned to name from the network's param_names. A law must reach θ through Param; that is how the checker and the gradient estimators know which clocks depend on which parameters.

source
Concourse.Mark — Type
Mark(name::Symbol)

The mark called name on the job the law applies to. Marks are per-job values drawn once at job creation (see MarkLaw). A service law that reads Mark(:size) gives each job its own service distribution.

source
Concourse.Enab — Type
Enab()

The wall-clock time at which the clock being built was enabled. This is the only time a law may read. In a generalized semi-Markov process (GSMP) a clock's distribution is frozen at its enabling, so "current time" has no meaning inside a law. Use Enab for time-varying rates, for example a nonhomogeneous arrival process:

Law(:Exponential,
    scale = inv(Param(:lambda) * (Const(1.0) + Const(0.5) * exp(Const(-0.02) * Enab()))))
source
Concourse.Const — Type
Const(value)

A literal number as a ScalarExpr. Bare numbers in arithmetic with expressions are promoted to Const automatically, so Param(:mu) * 2 and Param(:mu) * Const(2.0) are the same expression.

source
Concourse.InService — Type
InService(station::Symbol)

The number of jobs currently in service at the station named station, as a Float64 count. Occupancy is live state, not a value frozen at enabling: only the service law of a station may read it (compile check C5), and whenever the count changes the law is re-evaluated mid-flight, with the job's accrued service effort carried over and te left at the original enabling — the same segment convention processor sharing uses.

# A server that speeds up with the crowd, saturating at three helpers.
Law(:Exponential, scale = inv(Param(:mu) * min(InService(:cpu), Const(3.0))))
source
Concourse.InBuffer — Type
InBuffer(station::Symbol)

The number of jobs waiting in the buffer at the station named station, as a Float64 count. The rules of InService apply: only the service law of a station may read occupancy (compile check C5), and the law is re-evaluated whenever the watched count changes.

source

Laws

Concourse.Law — Type
Law(family::Symbol; kwargs...)

A probability distribution written as data. family names a Distributions.jl type, and each keyword argument is one constructor argument, given as a ScalarExpr or a number. The keyword names are labels for the reader; the values are passed to the constructor positionally, in the order written. So write them in the constructor's positional order.

Because a Law is inspectable data, reads_params, reads_marks, and reads_time report exactly what it depends on, and compile can check the model statically.

Examples

Law(:Exponential, scale = inv(Param(:mu)))            # Exponential with mean 1/μ
Law(:Gamma, shape = Const(2.0), scale = Const(0.5) * inv(Param(:mu)))
Law(:Dirac, value = Mark(:size))                      # deterministic, per job
source
Concourse.Opaque — Type
Opaque(f; params=Symbol[], marks=Symbol[], time=false)

The escape hatch for laws the expression algebra cannot write: an arbitrary function made legal by declaring what it reads. f has the signature (θnamed, marks, te) -> distribution, where θnamed is a NamedTuple of the declared params, marks is the job's mark NamedTuple, and te is the enabling time. The declarations params, marks, and time tell the checker what f reads; the checker trusts them, so its conclusions about an Opaque law are unverified.

Example

Opaque((θ, marks, te) -> Exponential(1 / (θ.mu * marks.weight));
       params = [:mu], marks = [:weight])
source
Concourse.MarkLaw — Type
MarkLaw(; name = law, ...)

A named collection of laws, one per mark. Marks are drawn once at job creation, in the written order, and recorded, so replay reproduces them. A later law in the list may read an earlier mark of the same job. Attach a MarkLaw to a source with the mark keyword of source!.

Example

MarkLaw(size = Law(:Exponential, scale = Const(1.0)),
        class = Law(:Uniform, a = Const(0.0), b = Const(2.0)))
source

Inspection

These predicates report what an expression or law reads. The compile-time checks are built on them.

Concourse.reads_params — Function
reads_params(x) -> Set{Symbol}

The parameter names that a ScalarExpr or a law reads through Param. For an Opaque law it returns the declared names. compile checks every name against the network's param_names.

Example

julia> reads_params(Law(:Exponential, scale = inv(Param(:mu))))
Set{Symbol} with 1 element:
  :mu
source
Concourse.reads_marks — Function
reads_marks(x) -> Set{Symbol}

The mark names that a ScalarExpr or a law reads through Mark. For an Opaque law it returns the declared names. compile checks that every read mark is produced by some source.

Example

julia> reads_marks(Mark(:size) - Const(1.0))
Set{Symbol} with 1 element:
  :size
source
Concourse.reads_time — Function
reads_time(x) -> Bool

Whether a ScalarExpr or a law reads the enabling time through Enab. For an Opaque law it returns the declared time flag.

Example

julia> reads_time(exp(Const(-0.02) * Enab()))
true

julia> reads_time(Param(:mu))
false
source
Concourse.reads_state — Function
reads_state(x) -> Set{Symbol}

The stations whose occupancy a ScalarExpr or a law reads through InService or InBuffer. An Opaque law cannot read state — its callback never receives a state view — so it reads none. compile permits a nonempty set only in the service law of a station (check C5).

Example

julia> reads_state(Param(:mu) * min(InService(:cpu), Const(3.0)))
Set{Symbol} with 1 element:
  :cpu
